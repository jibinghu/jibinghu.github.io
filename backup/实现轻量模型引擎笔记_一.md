cudaMemcpy 和 cudaMemcpyAsync：

- `cudaMemcpy`：
一定阻塞 CPU 线程，直到拷贝完成才返回。

- `cudaMemcpyAsync`：
不阻塞 CPU，只把拷贝任务放进某个 CUDA stream 的队列里。


---

像 CPU 的计算和 IO？

- CPU：
同步 IO：read() → CPU 等
异步 IO：aio_read() → CPU 继续算

- CUDA：
cudaMemcpy → CPU 等 GPU
cudaMemcpyAsync → CPU 继续跑

一句话：区别在“调用方等不等”，不是在“干了没干”。

---

默认流和其他流串行，但其他流之间是并行的：

``` cuda
stream0: |---- memcpy ----|
stream1:                                 |---- kernel ----|
stream2:                                |---- kernel ----|
```
